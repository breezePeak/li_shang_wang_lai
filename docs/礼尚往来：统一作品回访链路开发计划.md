# 礼尚往来：统一作品回访链路开发计划礼尚往来：统一作品

## 1. 本轮需求范围

本轮只实现“作品回访”链路，不实现完播功能。

作品回访定义为：

```text
互动用户评论或点赞了我的作品
→ 找到该用户主页
→ 找到该用户最近一条非置顶作品
→ 检查该作品是否已经点赞
→ 未点赞：进入“点赞 + 评论”回访候选
→ 已点赞：整条回访跳过，不点赞、不评论
```

重要规则：

* 如果目标作品已经点赞过，本轮回访直接跳过，不再发表评论。
* 第一阶段仅做候选预览，不执行真实点赞和真实评论。
* 真实执行必须在候选、审批、dry-run、二次确认链路完成后，且每轮最多执行 1 条。
* 不开放批量、无人值守回访。

## 2. 现状说明

当前项目已有：

* 评论原地回复流程：`prepare → approve → dry-run → confirm-execute → execute`；
* 点赞事件扫描；
* 点赞来源用户主页定位初版；
* 用户最近非置顶作品查找；
* 目标作品点赞状态检查；
* 点赞回访候选预览初版。

当前缺失：

* 评论事件触发回访；
* 评论用户主页定位；
* 评论与点赞来源合并；
* 统一的 `visit_work` action；
* 目标作品评论输入框定位与发送；
* 回访 dry-run；
* 回访“点赞 + 评论”真实执行；
* partial_failed 状态处理。

## 3. 新增统一动作类型

新增：

```text
action_type = visit_work
```

`visit_work` 不等于 `like_work`，它代表一次完整的作品回访任务。

数据结构至少包括：

```json
{
  "actionType": "visit_work",
  "sourceEventTypes": ["comment", "like"],
  "sourceEventIds": [1, 2],
  "actorName": "互动用户昵称",
  "actorProfileUrl": "用户主页地址",
  "relation": "friend | mutual | unknown",
  "targetVideoId": "目标视频ID",
  "targetVideoUrl": "目标视频地址",
  "targetVideoTitle": "目标视频标题",
  "likeState": "not_liked | already_liked | unknown",
  "visitCommentText": "经审核的回访评论",
  "likeRequired": true,
  "commentRequired": true,
  "status": "planned | skipped | blocked | approved | dry_run_ok | execute_confirmed | succeeded | partial_failed",
  "reason": null,
  "previewOnly": true,
  "executeAllowed": false
}
```

## 4. 新增统一计划命令

新增命令：

```bash
npm run visits:plan -- --source all --json
npm run visits:plan -- --source comment --json
npm run visits:plan -- --source like --json
```

来源规则：

### `--source like`

* 读取 `event_type=like` 的新事件；
* 使用已有点赞通知定位逻辑获取互动用户主页；
* 找到用户最近非置顶作品；
* 检查目标作品点赞状态；
* 已点赞则跳过；
* 未点赞则生成回访候选。

### `--source comment`

* 读取 `event_type=comment` 的新事件；
* 从扫描数据或评论页面中取得评论用户主页 URL；
* 找到用户最近非置顶作品；
* 检查目标作品点赞状态；
* 已点赞则跳过；
* 未点赞则生成回访候选。

### `--source all`

* 合并 comment 与 like 两种来源；
* 同一用户指向同一目标作品时只生成一条 `visit_work`；
* `sourceEventTypes` 和 `sourceEventIds` 保存全部来源。

## 5. 第一阶段：回访候选预览

第一阶段只实现计划与预览，不执行真实操作。

对每个候选执行：

```text
识别互动用户
→ 获取用户主页
→ 获取最近非置顶作品
→ 打开目标作品
→ 检查点赞状态
```

处理结果：

### 未点赞

```json
{
  "status": "planned",
  "likeState": "not_liked",
  "previewOnly": true,
  "executeAllowed": false
}
```

### 已点赞

```json
{
  "status": "skipped",
  "likeState": "already_liked",
  "reason": "目标作品已点赞，本次回访跳过，不再评论。",
  "previewOnly": true,
  "executeAllowed": false
}
```

### 无法确认

```json
{
  "status": "blocked",
  "likeState": "unknown",
  "reason": "无法确认目标作品或点赞状态。",
  "previewOnly": true,
  "executeAllowed": false
}
```

## 6. 第二阶段：回访评论候选

仅对 `likeState=not_liked` 的 planned 项生成回访评论候选。

规则：

* 回访评论内容第一阶段必须由用户审核；
* Agent 后续可结合目标作品标题或摘要生成候选评论；
* 未经审核不得执行评论发布；
* 不允许使用“来回访了”“已赞已评”“互相支持”等明显机械互动话术。

新增命令建议：

```bash
npm run visits:prepare -- \
  --plan-item-id <id> \
  --comment-text "<回访评论>" \
  --json
```

## 7. 第三阶段：回访 dry-run

新增命令：

```bash
npm run visits:execute -- --action-id <id> --dry-run --json
```

dry-run 只验证：

* 用户主页匹配；
* 目标作品匹配；
* 当前点赞状态可确认；
* 当前点赞状态为 `not_liked`；
* 评论输入区域可定位；
* 不点赞；
* 不发送评论。

如果 dry-run 时发现已经点赞：

```text
status = skipped
reason = 目标作品已点赞，本次回访跳过，不再评论。
```

## 8. 第四阶段：单条真实执行

新增执行方式：

```bash
npm run visits:execute -- --action-id <id> --execute --max-items 1 --json
```

执行前必须满足：

* action 已审批；
* dry-run 已成功；
* 已完成 execute confirm；
* 目标作品当前仍是 `not_liked`；
* 回访评论文本已审核；
* 本轮最多执行 1 条；
* 该目标作品不存在历史成功回访记录。

执行顺序：

```text
再次检查点赞状态
→ 如果已点赞：skipped，整条停止，不评论
→ 如果未点赞：点击点赞
→ 确认点赞成功
→ 输入已审核评论
→ 发布评论
→ 确认评论成功
→ 保存 succeeded 与审计记录
```

## 9. 部分失败处理

如果发生：

```text
点赞成功
→ 评论发送失败
```

不得自动重新执行整条任务。

必须记录：

```json
{
  "status": "partial_failed",
  "likeStatus": "succeeded",
  "commentStatus": "failed",
  "reason": "点赞已成功，但回访评论失败，需要人工检查。"
}
```

此状态需要人工处理，不自动重试。

## 10. 防重复规则

必须实现：

* 同一个用户、同一个目标作品，同一轮只生成一次回访候选；
* 已存在成功 `visit_work` 的目标作品，不再重复回访；
* 目标作品当前已点赞，整条任务跳过；
* `partial_failed` 不自动重跑；
* 点赞状态无法确认时不得执行。

## 12. 测试要求

必须补充测试：

* like 来源能够生成 visit 候选；
* comment 来源能够生成 visit 候选；
* comment + like 同一用户合并为一条候选；
* 目标作品未点赞时输出 planned；
* 目标作品已点赞时输出 skipped，且不生成评论动作；
* 点赞状态未知时 blocked；
* dry-run 不产生真实点赞或评论；
* 未审批不得执行；
* 未 confirm-execute 不得执行；
* 每轮真实执行最多 1 条；
* 点赞成功但评论失败时记录 partial_failed；
* 已成功回访的目标作品不得重复执行。

## 13. 本轮不做内容

本轮明确不做：

* 完播；
* 批量真实回访；
* 无人值守自动循环；
* 已点赞作品继续评论；
* 自动生成并直接发布回访评论；
* 自动解除任何阻断条件。
